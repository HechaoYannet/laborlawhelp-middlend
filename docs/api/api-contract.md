# API Contract v1 (Markdown Single Source)

## 1. Scope and Version
- Base path: `/api/v1`
- Phase 1 auth mode: anonymous
- Phase 2 auth mode: JWT (compatible with case/session model)
- This file is the single source of HTTP contract truth.

## 2. Common Rules
| Item | Rule |
|---|---|
| Content-Type (request) | `application/json` |
| Stream response | `text/event-stream` |
| Time format | ISO-8601 (UTC) |
| ID type | UUID unless stated otherwise |
| Idempotency | `client_seq` + `session_id` unique window |
| Trace | `X-Trace-Id` optional from client, generated by server if absent |

## 3. Auth and Ownership
| Phase | Required Header | Server Behavior |
|---|---|---|
| Phase 1 | `X-Anonymous-Token` | Parse or issue anonymous owner |
| Phase 2 | `Authorization: Bearer <access_token>` | Validate JWT and bind user owner |

Ownership validation applies to all case/session endpoints.

## 4. Case and Session APIs

### 4.1 POST /cases
Create a case.

Request body:

| Field | Type | Required | Enum/Format | Notes |
|---|---|---|---|---|
| title | string | no | max 200 | Default `未命名案件` |
| region_code | string | no | e.g. `xian` | Default `xian` |

Response 201:

| Field | Type | Notes |
|---|---|---|
| id | uuid | case_id |
| owner_type | string | `anonymous` or `user` |
| created_at | string | ISO-8601 |

### 4.2 GET /cases
List cases for current owner.

Query:

| Field | Type | Required | Notes |
|---|---|---|---|
| status | string | no | `active` or `archived` |
| page | int | no | default 1 |
| page_size | int | no | default 20, max 100 |

### 4.3 GET /cases/{case_id}
Get case details.

### 4.4 POST /cases/{case_id}/sessions
Create session under case.

Response 201:

| Field | Type | Notes |
|---|---|---|
| id | uuid | session_id |
| case_id | uuid | parent case |
| status | string | `active` |
| openharness_session_id | string | nullable at creation |

### 4.5 GET /cases/{case_id}/sessions
List sessions of a case.

### 4.6 GET /sessions/{session_id}/messages
List messages by page.

Query:

| Field | Type | Required | Notes |
|---|---|---|---|
| cursor | string | no | opaque cursor |
| limit | int | no | default 30, max 100 |

### 4.7 PATCH /sessions/{session_id}/end
End session by user action.

Request body: empty.

Response 200:

| Field | Type | Notes |
|---|---|---|
| id | uuid | session id |
| status | string | `ended` |
| ended_at | string | ISO-8601 |

## 5. Chat Stream API

### 5.1 POST /sessions/{session_id}/chat
Request body:

| Field | Type | Required | Notes |
|---|---|---|---|
| message | string | yes | User input |
| attachments | array | no | Attachment list |
| client_seq | int | yes | Monotonic by session from client |

Attachment object:

| Field | Type | Required | Notes |
|---|---|---|---|
| id | string | yes | Attachment id |
| name | string | yes | File name |
| url | string | yes | HTTPS URL |
| mime_type | string | yes | e.g. `image/png` |

Response: `text/event-stream`

SSE sequence (normal path):
1. `message_start`
2. `content_delta` (0..n)
3. `tool_call` / `tool_result` (0..n)
4. `final`
5. `message_end`

### 5.2 SSE Event Schema

`message_start`

```json
{"message_id":"msg_xxx"}
```

`content_delta`

```json
{"delta":"...","seq":13}
```

`tool_call`

```json
{"tool_name":"intent_router","args":{"topic":"termination"}}
```

`tool_result`

```json
{"tool_name":"intent_router","result_summary":"classified as unlawful termination"}
```

`final`

```json
{"message_id":"msg_xxx","summary":"...","references":[],"rule_version":"v2.2"}
```

`message_end`

```json
{"message_id":"msg_xxx"}
```

`error`

```json
{"code":503,"message":"服务暂时不可用，您可稍后重试。","retryable":true}
```

## 6. Auxiliary APIs
| Endpoint | Method | Purpose |
|---|---|---|
| `/sessions/{session_id}/summary` | GET | Structured summary |
| `/sessions/{session_id}/document` | GET | Download generated doc |
| `/cases/{case_id}/triage` | GET | Case triage result |

## 7. Phase 2 Auth APIs
| Endpoint | Method | Purpose |
|---|---|---|
| `/auth/sms/send` | POST | send SMS code |
| `/auth/sms/login` | POST | login with SMS code |
| `/auth/refresh` | POST | refresh token |
| `/auth/logout` | POST | revoke refresh token |

## 8. Compatibility Rules
- Keep case/session model unchanged across phases.
- Unknown response fields must be ignored by clients.
- New SSE event types must not break existing parser (default ignore unknown event).
